# 照片复刻桌宠与温柔互动反馈设计（P1）

## 目标

在已经可拖动、可收起到桌面的透明桌宠上，替换通用白兔形象，并加入一次性、低干扰的直接互动反馈。新形象以用户提供的真实兔子照片为唯一角色参考：白色蓬松毛、左眼黑色眼圈、粉色耳内侧、圆脸与小粉鼻子。

本阶段的成果是一只视觉上具有 3D 体积感、但不使用实时 3D 引擎的照片复刻型桌宠；它趴在桌面上，点击时以温柔方式回应。

## 范围与非目标

### 本阶段范围

- 准备一张透明背景的“趴着待机”写实 3D 风格兔子资源，作为桌宠默认形象。
- 预留“被摸摸”和“进食”两个同风格姿态资源的加载接口；素材尚未提供时，复用待机图而不显示空白。
- 点击桌宠触发一次 `petted` 养成事件，并显示一个空心爱心上浮。
- 通过右键菜单喂食时显示一颗小胡萝卜滑入；休息时只更新既有养成状态，不播放循环动画。
- 保持现有拖动、双击打开学堂、右键菜单与位置持久化行为。

### 非目标

- 不在本阶段实现真实扫描、可自由旋转的 GLB 模型、骨骼绑定或自由走动。
- 不引入常驻游戏循环、定时轮询、音频、语音或全天候状态动画。
- 不调整学堂课程、属性、存档数据格式和小游戏主线。

未来如果用户补充左右侧、背面照片或短环绕视频，可在不改变本阶段交互接口的前提下，将资源替换为完整 GLB 模型。

## 视觉与交互

### 默认桌宠

- 窗口继续为透明、无边框、始终在前的 `NSPanel`，默认内容尺寸保持 260×300 pt。
- 默认资源呈现兔子趴坐在桌面边缘的姿态，只保留角色及必要的细微接地阴影；窗口其余像素完全透明。
- 写实程度以照片相似度优先，不采用泛用卡通白兔的脸型或花纹。

### 温柔陪伴反馈（用户选择 A）

- 单击：在兔子上方浮出一个空心爱心，轻微上移并在 0.8 秒内淡出；同时派发既有 `.petted` 操作。
- 喂食：一颗小胡萝卜从边缘滑入角色嘴边，随后淡出；同时派发既有 `.fed` 操作。
- 休息：派发既有 `.rested` 操作，不叠加循环或明显动画。
- 双击优先于单击：双击仅打开学堂，不应先显示摸摸爱心。
- 右键不触发摸摸反馈，只展示菜单。

## 架构与数据流

1. `PetVisualAsset` 负责定位、解码并按用途选择透明角色资源。它优先返回目标姿态；找不到时返回待机姿态；待机也缺失时由现有安全占位路径处理。
2. 新增纯值类型 `DesktopPetFeedback`，表达 `.pet`, `.feed` 或无反馈，并从用户动作映射为图标、文案和持续时间。该类型不依赖 AppKit 或 SwiftUI，便于单测。
3. `DesktopPetView` 继续持有 `PetStore`。它先判定手势意图，再在单击或菜单动作中同时调用 `PetStore.dispatch` 与一次性反馈控制器。
4. 反馈控制器只保存当前反馈与一个可取消的 `DispatchWorkItem`；新操作会取消旧的清理任务并替换画面，超时后清空状态。
5. `DesktopPetWindowController` 无需理解反馈细节，仍只管理面板生命周期和位置。

## 低资源约束

- 桌宠空闲时为单张静态透明图：无 `Timer`、无逐帧序列、无实时 3D/WebGL、无持续状态轮询。
- 爱心与胡萝卜只在用户操作时出现一次；其移位与淡出交给 SwiftUI 合成动画，状态清理由一个可取消的延迟任务完成。
- 同一时刻至多保留一个反馈覆盖层与一个清理任务，连续点击不会累计视图或任务。
- 新图像资源控制在适合 260×300 pt 显示的尺寸，避免把多张原始高分辨率照片常驻解码。

## 测试与验收

- `DesktopPetFeedback`：验证摸摸、喂食、休息、双击和右键的反馈映射及持续时间。
- 资源选择：验证待机资源是必需回退，缺少可选姿态时不会让桌宠空白。
- 既有回归：双击打开学堂、单击摸摸、菜单喂食/休息、位置持久化相关测试继续通过。
- 手工验证：拖动桌宠后，单击出现一次爱心；右键喂食出现一次胡萝卜；快速连续点按不残留多个元素；双击不闪出爱心；60 秒静置无持续动画。
- 性能验收：在用户 Mac 上用 Instruments 记录 60 秒静置 CPU 与 RSS，目标保持 P0 的空闲 CPU 平均低于 1%，RSS 以 120 MB 为警戒线。

## 风险与降级

- 现有照片没有完整环绕角度，生成资源应以正面/半侧趴坐为主，不宣称真实扫描精度。
- 写实素材若没有足够相似度，不覆盖现有资源；先让用户在运行中的桌宠上确认后再替换默认图。
- 图片解码失败时，保留可交互桌宠与学堂入口，避免将整个窗口渲染为空白。
