# 透明常驻桌宠设计（P0）

## 目标

把小白从学堂窗口中的展示图，变成能长期停留在 macOS 桌面上的透明、可拖动、低干扰桌宠。学堂继续作为完整养成界面；桌宠是随时可见的轻量入口。

## 桌面模式

- 使用独立的 `NSPanel` 承载桌宠，初始内容尺寸为 260×300 pt。
- 面板无标题栏、无阴影、非不透明，窗口背景透明，层级为 floating。
- 面板显示在所有 Space 与全屏应用辅助层；不抢占应用焦点。
- 拖动非交互区域可移动桌宠；窗口坐标保存到 `UserDefaults`，启动时尽量恢复到上次位置。
- 双击小白打开或聚焦学堂窗口。
- 右键菜单提供：喂食、休息、打开学堂、退出。
- 单击小白仅触发本地“摸摸”反馈和既有 `petted` 事件；不打开学堂。

## 视觉资产

- 桌宠使用独立的透明 PNG 主形象，而非当前带米色背景的学堂主图。
- 首版使用 3D 风格静态形象，加呼吸、眨眼与极轻方向摆动；它不是骨骼绑定模型。
- 若透明资产缺失，桌宠模式不显示带背景的矩形替代图，以免破坏桌面融合感；改为保留学堂模式并显示可诊断的占位标志。

## 低资源约束

- 空闲时不得存在逐帧游戏循环、`Timer` 或持续轮询。
- 呼吸和摆动使用 Core Animation/SwiftUI repeat 动画，由合成器处理；`accessibilityReduceMotion` 开启时禁用连续动画。
- 眨眼以单次 `DispatchWorkItem` 低频调度；窗口离开屏幕或应用不活跃时取消待执行任务。
- 小游戏和学堂只在用户显式打开时加载；桌宠面板不创建课程、房间或小游戏视图。
- 性能验收目标：空闲 CPU 平均低于 1%，常驻内存以 Instruments 的实际 RSS 记录为准，并以 120 MB 作为警戒线而非保证值。

## 架构

1. `DesktopPetWindowController` 只负责创建、显示、隐藏、移动和持久化 `NSPanel`。
2. `DesktopPetView` 只渲染透明角色、手势与菜单，接收 `PetStore`。
3. 现有 `AcademyView` 继续担任完整学堂界面；它不再承担紧凑桌宠布局。
4. `PetPosition` 是可编码的 `x/y` 值；恢复时会被限制在可见屏幕范围内。
5. 用户事件仍流向现有 `PetStore.dispatch`，不新增第二套存档。

## 测试与验证

- 纯逻辑测试：位置编码/解码、屏幕边界钳制、单击/双击事件映射。
- 资源测试：透明桌宠资源存在且可解码，包含 alpha 通道。
- 构建测试：完整 Swift 测试通过。
- 人工验收：桌宠可拖动、重启后位置恢复、双击能打开学堂、低动态效果时无循环动画。
- 性能验收：使用 Instruments 的 Time Profiler 与 Allocations 测量 60 秒静止状态，记录 CPU 与 RSS。

## 非目标

- 自动巡游、物理碰撞、跨应用内容识别。
- 真正的 3D 骨骼模型或高帧率序列帧。
- 开机自启、签名、公证；这些属于 P4。
