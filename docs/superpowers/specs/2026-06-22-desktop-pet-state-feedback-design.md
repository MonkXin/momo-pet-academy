# 桌宠低干扰状态反馈设计

## 目标

让桌宠把既有饥饿、疲惫、低心情状态呈现为一次性、低资源反馈，不新增轮询或持续动画。

## 呈现规则

- 优先级：疲惫 > 饥饿 > 想摸摸 > 待机。
- 疲惫：使用透明闭眼休息姿态；不显示闪烁提示。
- 饥饿：保持待机姿态，仅在首次进入状态时显示一次短暂胡萝卜提示。
- 想摸摸：保持待机姿态，仅在首次进入状态时显示一次短暂爱心提示。
- 状态恢复后回待机。单击摸摸、右键喂食等即时反馈优先，结束后再显示当前状态。

## 架构与验证

- 使用独立的纯值类型映射 `PetProfile` / `PetActivity` 到姿态与一次性提示意图。
- `DesktopPetView` 仅观察映射变化并复用现有可取消的短时反馈机制。
- 单测覆盖优先级、休息姿态、状态恢复与提示去重；人工验证连续操作与 60 秒静置。
