# 奶茶首版发布体验设计

## 目标

将宠物的可见名称统一为“奶茶”，并完成可日常使用的首版启动体验：启动即为桌宠、可双击进入学堂、可手动运行发布构建。

## 命名

- 应用名为“奶茶的学堂时光”。
- 学堂、桌宠、小屋、成长日记、无障碍标签和测试中的可见“小白”统一为“奶茶”。
- 图片资源、模块名、存档键与内部文件名保持不变，避免破坏现有构建和用户存档。

## 启动与发布

- 读取存档后默认显示桌宠，不自动打开学堂大窗口。
- 首次启动显示一次静态提示：双击奶茶打开学堂；右键可照料或退出。
- 保留手动运行发布构建的入口；发布前执行完整测试和 release build。
- 登录启动作为可选开关，默认关闭；失败时保留手动启动，不做后台重试。

## 性能与验收

- 桌宠启动和待机不创建小屋或学堂重视图。
- 发布验收以静置 CPU 约 0.1%、GPU 0、闲置唤醒接近 0 为基线。
- 旧档打开后奖励、摆放家具、桌宠位置和进度均保持。
